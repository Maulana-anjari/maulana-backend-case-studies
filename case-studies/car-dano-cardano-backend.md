# CAR-dano / Cardano Backend

## Repository

- Original repo: https://github.com/CAR-dano/cardano-backend

## Overview

Backend system for a blockchain-backed vehicle inspection platform on Cardano.

The work focused on making the backend production-ready, fast, and reliable while supporting inspection workflows and blockchain verification.

## My Role

- Backend Engineer
- Focused on backend architecture, API modules, database design, infrastructure, and production integration

## Problem

The platform needed a backend that could support real inspection workflows, Cardano-based verification, and day-to-day operational reliability for concurrent users.

## Stack

- Cardano
- Node.js
- Prisma
- PostgreSQL
- REST APIs
- CI/CD
- NFT minting

## Architecture / Flow

- REST API layer for inspection and platform workflows
- Database layer for transactional and operational data
- Cardano integration for blockchain-backed verification
- Deployment and CI/CD for production reliability

## Impact

- Reduced vehicle-inspection turnaround by up to **50%**
- Kept the platform stable with **~100% faster response time** and **100% uptime** for **20 concurrent users**

## Challenges

- Balancing blockchain verification with practical backend performance
- Keeping inspection workflows stable under real usage
- Supporting production changes without breaking existing behavior

## What I Learned

- Great backend systems make blockchain features usable in practice
- Operational simplicity matters more than technical novelty
- Performance work often compounds across the whole product
